# 트러블슈팅 (Troubleshooting)

Open-RMF 오피스 데모 관련 문제/해결 기록.
환경: **Ubuntu 24.04 + ROS 2 Jazzy + Gazebo Harmonic(gz-sim 8)**

> 셋업/실행 전체 흐름은 [README.md](./README.md) 참고.

---

## rmf_demos_maps 빌드 실패 — `building_map_generator: invalid choice: 'ignition'`

**증상**
- `colcon build`(특히 clean 빌드) 시 `rmf_demos_maps`만 실패:
  ```
  building_map_generator: error: argument command: invalid choice: 'ignition'
    (choose from 'gazebo', 'nav', 'navgraph_visualization')
  gmake: *** [maps/campus_ign/campus.world] Error 2
  Failed   <<< rmf_demos_maps
  ```
- `campus` / `airport_terminal` / `battle_royale` 등의 `*_crowdsim_ign` 타겟에서 터짐.

**원인**
- `rmf_demos_maps/CMakeLists.txt`가 맵마다 **`_ign`(옛 Ignition Gazebo) 월드**를
  `ros2 run rmf_building_map_tools building_map_generator ignition ...` 로 생성하려 함.
- 그런데 Jazzy의 apt `rmf_building_map_tools`는 **`ignition` 서브커맨드를 제거**함
  (Ignition → Gazebo 개명으로 `gazebo`에 통합).
- → 유효하지 않은 선택지라 에러. 게다가 `_ign` 월드는 **Jazzy(새 Gazebo)에선 쓰지 않음**.

**해결** — `CMakeLists.txt`에서 `_ign` 블록 삭제
- 파일: `rmf_ws/src/rmf_demos/rmf_demos_maps/CMakeLists.txt`
- `## Ign worlds` 섹션부터 `generate_..._crowdsim_ign` 타겟 +
  `install(DIRECTORY ${ign_output_dir} ...)` 까지 한 블록을 통째로 제거.
- (최소 수정으로 그 블록의 `ignition` → `gazebo` 로 바꿔도 빌드는 통과하지만,
  안 쓰는 월드를 중복 생성하므로 삭제가 깔끔함.)
- 이후 `colcon build` → 정상 통과 (office.world 재생성됨).

**⚠️ 주의 — 클론한 소스를 수정한 것**
- 이 패치는 `rmf_demos`(외부 repo) 안의 파일을 고친 거라, 그 clone의 git에서 **modified**로 잡힘.
- **`git checkout .` 같은 광범위 되돌리기 / rmf_demos 재clone** 하면 패치가 날아가 **에러 재발** → 그땐 다시 이 블록 삭제.
- 특정 파일(예: `office.building.yaml`)만 되돌릴 땐 `git checkout -- <그 파일만>` 으로 콕 집어서.
